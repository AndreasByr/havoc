<objective>
Close the one blocking gap from Phase 3 verification: the Matrix-Bot internal HTTP server silently
accepts all requests when BOT_INTERNAL_TOKEN is empty, violating the "Fail Loud, Never Fake" project
security constraint.

Purpose: The Matrix-Bot's auth guard wraps the entire authentication block in
`if (token && token.length > 0)`, which means an unconfigured (empty) token causes the auth check to
be skipped entirely and every request is accepted. The Hub's equivalent (`requireInternalToken` in
`internal-auth.ts`) correctly throws 503 when the token is unconfigured. The Matrix-Bot must mirror
this behavior.

Output: A corrected `internal-sync-server.ts` where the auth check is unconditional (fail-loud 503
when token is empty, 401 when token is wrong), and a new test case covering the 503 MISCONFIGURED
path.
</objective>
