## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Admin request → install/uninstall routes | requireSuperadminSession / requireAdminSession enforced before route body; only admins reach the log lines |
| stdout | Audit log goes to docker stdout — readable by container host admin; not exposed externally |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Information Disclosure | audit log stdout | accept | Log contains appId and event type only — no secrets, no PII, no install URL; acceptable for admin-facing docker logs |
| T-02-08 | Repudiation | app lifecycle events | mitigate | Structured log with appId + event creates an auditable trail for install/uninstall actions; sufficient for solo-admin use case |
| T-02-09 | Tampering | audit log format | accept | console.log(JSON.stringify) is tamper-evident at the process level; no integrity hash needed for this threat model |
</threat_model>
