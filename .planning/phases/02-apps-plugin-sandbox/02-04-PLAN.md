<objective>
Set process-level memory caps for the two app-code-executing Node.js processes per D-05 and D-06:
- Bot: `--max-old-space-size=512` added to the `start` script in `package.json` (production + local non-Docker)
- Hub: `--max-old-space-size=1024` added to both `Dockerfile` CMD (production Docker) and `package.json` `start` script (local preview)

Document `APP_HOOK_TIMEOUT_MS` in `.env.example` under a new `Apps / Plugin System` section.

The `dev` script is intentionally left unchanged in both packages тАФ memory limits are less critical during development and adding `NODE_OPTIONS` to `dev` would affect tsx/nuxt-dev process trees unnecessarily.

Purpose: A memory-leaking or memory-bombing app can crash the entire bot/hub process. The V8 heap cap bounds the damage to a process OOM (which produces an error log) rather than gradual host starvation.
Output: Four modified config files; all changes are 1-3 line additions, no logic changes.
</objective>
